# Response formatting rules

When you answer in Persian or mixed Persian and English, follow these rules strictly:

1. Do not write mixed Persian and English in the same paragraph unless necessary.
2. If you mention code, file names, commands, paths, variables, or technical terms, always put them inside backticks.
3. For long answers, use this structure exactly:
   - First: a very short English heading for the section
   - Then: Persian explanation in separate lines
4. Never produce one sentence that contains long Persian text mixed with raw English identifiers.
5. For commands and code, always use fenced code blocks.
6. For filenames, paths, setting keys, extensions, and UI labels, always wrap them in backticks.
7. Keep Persian sentences short.
8. Prefer numbered steps instead of dense paragraphs.
9. If the answer is mostly technical, prefer English for the technical content and Persian only for explanation.
10. Do not use tables unless explicitly requested.
